# AI绘画提示词提取器

本skill用于分析上传的图片或视频，也用于把用户的文字想法结构化成可生成的AI绘画提示词，并能复盘生成结果与目标之间的差距。通过内置的海量风格词库和画面锚点流程，确保提示词既专业、精准，又适合小白直接复制使用。

本skill已经内置稀有亚种风格词库与新东方神怪水墨语法。使用时不要把它当成随机风格生成器；它的目标是先准确拆解用户图片，再用更细的风格词解释“这张图像哪里像、该怎么写进提示词”。

## 全局输出默认

- 默认使用中文输出分析、解释、标题和提示词内容；除非用户明确要求英文或双语，不要输出纯英文提示词。
- 用户未指定平台时，默认把"直接复制提示词"适配为 GPT Images / ChatGPT Images 使用的自然语言描述；不要默认输出 Midjourney 参数、版本号或 `--ar`、`--v`、`--q` 等命令式参数。
- GPT Images 版提示词优先使用完整句子描述画面、构图、主体、光影、材质、细节边界和排除项；可以保留少量必要英文风格术语，但必须用中文主导。
- GPT Images 版必须单独写明镜头方向/相机轴线：相机从主体哪一侧、哪一高度看，脸朝画面哪边，视线是否看向镜头。禁止只写"特写、近景、俯视"而不写相机方向。

## 通用风格骨架输出默认

拆解提示词时，必须先区分"稳定可迁移层"和"可随机创作层"。稳定可迁移层用于复用画面质感、皮肤质感、色彩质感、光影方式和审美方向；可随机创作层用于角色身份、具体服装款式、道具、场景细节和装饰变化。不要把人物主体描述当成风格拆解的全部。

默认按以下维度输出中文提示词，每个维度尽量给出可直接复制的短语：

- 画面质感：渲染方式、清晰度、边缘处理、厚涂/赛璐璐/版画/胶片/水晶感等。
- 色彩质感：饱和度、冷暖关系、柔和程度、主色与点缀色的分布方式。
- 皮肤质感：肤色、光泽、纹理保留程度、阴影边界、是否弱化真实毛孔。
- 人物审美：脸型、五官比例、年龄感、气质、二次元/写实/浮世绘化程度。
- 服饰材质：布料、丝绸、皮革、金属、刺绣、漆面、透明或反射材质。
- 动作姿态：身体方向、头部角度、手势、遮挡关系、动态趋势。
- 视觉重点：唯一锚点、次级锚点、哪些元素必须弱化，防止抢焦点。
- 光线方式：光源方向、软硬、阴影形状、高光边界、焦散/反射/逆光方式。
- 互动对象：人物与花、刀、宝石、镜面、水、球体、文字等对象的空间关系。
- 人物状态：表情、情绪、凝视方向、松弛/压迫/神秘/冷静等状态。
- 场景氛围：暗场、留白、浮世绘背景、潮湿空气、梦幻、神秘、华丽等。
- 文字限制：是否允许可读文字、文字大小位置、是否禁止水印/logo/乱码。

当用户需要"找自己的路"或分析多张图时，优先输出跨图共同稳定词，再列出可以保留随机性的角色、服饰、道具和场景变量。

## 何时调用

- 用户上传图片或视频文件
- 用户要求"分析这张图片的风格"
- 用户要求"提取这张图片的提示词"
- 用户要求"生成这张图片的AI绘画提示词"
- 用户提供图片URL并请求提示词分析
- 用户没有参考图，只描述想要的画面、角色、风格或氛围
- 用户要求"帮我写一个直接复制的提示词"
- 用户提供生成结果并要求"差距在哪"、"复盘一下"、"改到更像目标"

## 工作流程

### 1. 接收输入

**图片文件**:
- 支持格式: JPG, JPEG, PNG, GIF, BMP, WEBP
- 使用视觉能力读取图片内容

**视频文件**:
- 支持格式: MP4, MOV, AVI, MKV, WEBM
- 提取关键帧（每5秒提取一帧，最多10帧）
- 对每个关键帧单独分析

**文字想法**:
- 用户可能只提供模糊需求，例如"想要一个浮世绘白发少女"
- 不要因为信息不完整而频繁追问；优先补合理默认值并明确标注
- 只有当缺失信息会显著改变结果时，最多先问一个关键问题

**生成结果复盘**:
- 用户可能同时提供目标图、生成图、提示词或只提供生成图
- 对比时先找"为什么不像"，再重写可复制提示词

### 2. 判断工作模式

先判断任务属于哪种模式，然后按对应优先级生成结果：

| 模式 | 触发场景 | 目标 | 优先级 |
|------|----------|------|--------|
| 图像拆解模式 | 有参考图/视频，要求提取或复刻提示词 | 准确还原图像核心视觉 | 构图 > 姿态 > 主体位置 > 视觉主次 > 可见物件微细节 > 材质与光效 > 风格 |
| 想法构建模式 | 没有参考图，只有文字想法 | 把模糊想法补成完整画面规格 | 主体 > 动作 > 场景 > 镜头 > 构图 > 光影 > 风格 > 负面约束 |
| 差距复盘模式 | 有生成结果，要求分析差距或重写 | 找出跑偏原因并给修正版 | 目标差异 > 跑偏原因 > 锁定锚点 > 负面约束 > 修正版提示词 |

### 3. 画面骨架与锚点

在生成提示词之前，必须先建立画面骨架。画面骨架用于防止提示词只堆风格词、却丢失用户真正想要的画面。

**画面骨架字段**:
- 主体：人物/动物/物体/场景是什么，数量是多少
- 动作/状态：姿态、表情、手部动作、身体方向、运动趋势
- 场景：背景、环境、时代、空间关系
- 镜头：远景/全身/半身/中近景/特写，俯视/仰视/平视，广角/鱼眼/长焦；必须单独写镜头方向/相机轴线（从左前方、右前方、侧面、正面、上方或下方看）
- 构图：中心/偏置/对角线/旋转/三分法/黄金分割/紧裁切
- 光影：光源方向、软硬、对比度、高光与阴影位置
- 色彩：主色、辅色、点缀色、饱和度、冷暖关系
- 材质：丝绸、金属、玻璃、皮革、毛发、水面等
- 风格：艺术流派、文化风格、数字风格、艺术家参考
- 质量与平台：Midjourney、Stable Diffusion、Flux、DALL-E 3等

**不可变锚点**:
- 图像拆解模式：提取6个不可变锚点，包括视角、裁切、姿态、脸部/主体位置、手部/关键动作、主体与背景主次
- 想法构建模式：从用户文字中识别最重要的3-5个意图锚点，不足时补默认值
- 差距复盘模式：把目标与生成结果的差异转成下一版必须锁定的锚点
- 不允许风格词覆盖锚点；复刻类任务中，姿态和构图永远比"华丽、精致、超详细"更重要

**可自由发挥区域**:
- 明确哪些元素可以让模型自由发挥，例如小配饰、背景纹样、局部花纹、气氛细节
- 对容易跑偏的元素加边界词，例如 limited, controlled, subtle, not overcrowded

### 3.1 图像拆解保真闸门

当用户提供参考图并希望复刻、靠近、拆解提示词，生成提示词前必须先通过本闸门。禁止只写"高级感、清透、华丽、发光、玻璃质感"这类总括词；必须把画面中已经可见的结构和微细节转成可生成语言。

**强制清点顺序**:
1. 画面比例与留白：宽高比、主体占画面比例、黑场/白场/背景留白比例、裁切位置。
2. 主体姿态：头部是否歪斜、脸朝向、眼睛露出数量、下巴/脖子/肩膀是否出现、身体是否被黑暗吞没。
3. 前景动作：手掌方向、手指数量与弯曲状态、拇指位置、手与发光物是否接触、手部透视是否夸张。
4. 锚点物件：锚点是什么、大小、位置、是否遮脸、与眼睛/手/光线的连接关系。
5. 锚点微细节：若是花，必须写花瓣数量范围、花瓣层次、花蕊颜色和数量、花瓣边缘形状、透明度、内部纹理、碎片、露珠、星芒、折射边；若是眼睛，必须写虹膜颜色、瞳孔、反射图案、高光形状、光束方向；若是手，必须写指尖、指甲、掌纹、关节、皮肤光泽。
6. 材质与光效：透明/半透明/镜面/水晶/玻璃/金属/丝绸等必须对应到具体物件，不能只放在风格词里。
7. 细节密度分区：哪里高密度、哪里低密度、脸部是否干净、背景是否空。

**缺失审计**:
在输出提示词前，逐项检查是否遗漏：
- 核心物件的形状细节（例如花瓣、花蕊、叶片、碎片）
- 核心材质的生成机制（透明度、折射、边缘高光、内部纹理）
- 人物五官细节（瞳孔、虹膜、高光、嘴唇、表情）
- 人物脸型几何（脸颊宽度、下颌线弧度、下巴长度和形状，是否为尖V脸）
- 姿态细节（歪头、遮挡、脖子/肩膀是否出现）
- 前景动作细节（手型、指尖、掌心、接触/悬浮关系）
- 视觉主次（唯一锚点、禁止竞争锚点、背景留白）

若任一项在参考图中明显可见但提示词未覆盖，必须先补齐再输出。复刻类任务的最低标准是：再次生成时允许脸型、发型、具体形态变化，但不允许缺失参考图中最显眼的细节、材质机制和光效关系。

### 3.1.1 脸型几何锁定

二次元人物不能只写"二次元脸、精致脸、干净脸"。模型默认会把女性角色美化成尖下巴、窄下颌、V形脸；如果参考图是圆润、短下巴、紧凑椭圆脸，必须把脸型当作几何锚点写入提示词。

**必须观察并写明**:
- 脸轮廓：圆润椭圆、紧凑椭圆、心形、V形、长脸、方圆脸。
- 脸颊：是否有柔和圆弧、脸颊是否比下巴更宽、是否有轻微饱满感。
- 下颌线：是柔和U形过渡，还是明显V形收窄。
- 下巴：短而圆、小而钝、尖长、过度突出；不要只写"小下巴"。
- 五官位置：眼睛到下巴的距离是否短，嘴巴是否靠近下巴，鼻唇是否弱化。
- 视角影响：俯视会压短下半脸，近景会放大脸颊曲线；必须和镜头一起描述。

**圆润二次元脸的稳定写法**:
```
圆润紧凑的二次元脸型，柔和饱满的脸颊弧线，短而小的圆钝下巴，U形下颌过渡，弱化鼻梁和嘴唇体积，不要尖V脸，不要长窄脸，不要尖下巴
```

**圆润但非幼态的稳定写法**:
```
圆润但非幼态的年轻成人二次元脸，脸颊柔和但不是婴儿肥，眼睛不要过大过圆，保留细窄鼻梁阴影和轻微嘴角线，短圆下巴但下半脸不过度缩短，不要儿童脸，不要幼态娃娃脸，不要低龄萌系比例
```

**常见失败与修正**:
- 生成成尖下巴美少女：新增"短圆下巴、U形下颌线、脸颊宽度保留、不要V形下颌、不要尖长下巴"。
- 生成成成熟写实脸：新增"图形化二次元面部、弱化真实颧骨和鼻唇体积、不要写实高颧骨、不要时尚模特脸"。
- 俯视图下脸被拉长：新增"俯视压短下半脸，眼睛到下巴距离短，嘴巴靠近短圆下巴"。
- 生成过度幼态：新增"年轻成人比例、眼睛不要过大过圆、脸颊不是婴儿肥、保留细窄鼻梁阴影和嘴角线、不要低龄萌系比例"。

### 3.1.2 手部框景与前景手势锁定

当参考图里的手指贴近镜头、从边缘伸入、夹住脸部/物体、形成取景框或遮挡边界时，手不是普通动作，而是构图骨架。禁止只写"手靠近脸、戴手套、手放在前景"；必须写清每只手的进入边、手掌方向、可见手指数、指尖位置、遮挡关系和负空间形状。

**必须观察并写明**:
- 手的数量和左右：左手/右手分别从画面哪个边缘进入。
- 手掌方向：掌心朝镜头、手背朝镜头、侧面手指、戴手套或裸手。
- 手指数与位置：每只手露出几根手指，哪些手指贴近眼睛/脸颊/前景物，哪些被裁切。
- 框景关系：手指是否围出三角形、C形、V形、圆框；框住的是脸、眼睛、镜片、花朵还是发光物。
- 遮挡层级：手指在脸/物体前方还是后方，遮住哪一部分，不能漂浮到错误层。
- 边缘裁切：手腕、手指尖、手掌是否被画面边缘裁掉。

**稳定写法**:
```
戴白色手套的双手作为贴近镜头的前景框架，手指从画面左右边缘伸入，围住脸部和头盔镜片；左侧手指贴近左眼外侧，右侧三根手指横跨右眼和脸颊前方，手指在人物脸前方形成C形/取景框式负空间，指尖被边缘局部裁切，不要随机手势，不要多余手指，不要把手放到背景后方
```

**常见失败与修正**:
- 生成随机比心/剪刀手：新增"不是比心手势、不是胜利手势、不是普通摆拍手势，而是前景取景框手势"。
- 手指漂到错误层级：新增"手指在脸部和镜片前方，遮挡脸颊/眼睛边缘，不能在头发或背景后方"。
- 手数量或手指错误：新增"两只手，各自可见指定数量手指；禁止三只手、多余手指、缺失手指、融合手指"。
- 手破坏主体：新增"手是边框，不是最大主体，不要遮住两只眼睛和核心反射物"。

### 3.1.3 虹膜灵魂与瞳孔微结构锁定

当参考图的眼睛是视觉核心时，不能只写"彩虹眼、发光眼、高饱和虹膜"。模型会生成漂亮但空泛的渐变眼，缺少瞳孔中心、内外环、星芒、反射符号和视线灵魂。必须把虹膜当作微型画面拆解。

**必须观察并写明**:
- 虹膜外环：深色外圈、彩色外圈、是否有粗细变化。
- 虹膜内环：瞳孔周围是否有金色/黄色/粉色核心环。
- 瞳孔：圆形、竖瞳、黑色小圆点、机械孔、星形孔、是否偏心。
- 放射纹：从瞳孔向外的细线、齿轮状纹、花瓣状纹、星芒线。
- 高光：白色圆点、弧形高光、十字星芒、镜片反射光点的位置和数量。
- 反射图案：是否有宇宙、花朵、窗光、彩虹光斑映在虹膜内。
- 两眼差异：左右眼是否同色、角度是否不同、主眼是否更亮。

**稳定写法**:
```
虹膜不是普通彩虹渐变，而是有灵魂的多层微结构：深紫外环，金黄色瞳孔内环，黑色清晰瞳孔，放射状细线和星芒纹，白色圆形高光与小十字光点，虹膜内有微小彩虹反射和宇宙光斑，左右眼都清楚可见，不要空洞渐变眼，不要普通糖果色眼睛
```

**常见失败与修正**:
- 生成只有彩虹颜色但空洞：新增"清晰黑色瞳孔、金色内环、深色外环、放射状纹理、星芒高光"。
- 眼睛变幼态：新增"眼睛不要过大过圆，保留椭圆眼型和清晰上眼睑压线"。
- 高光糊成一片：新增"高光是分离的圆点和小十字星芒，不是整片白色糊光"。

### 3.1.4 斑驳硬光与彩虹焦散锁定

如果参考图的真实感来自手指、镜片、玻璃或金属投下的斑驳光影，不能只写"过曝、彩虹光斑、镜头光晕"。光影必须被写成结构：光源方向、遮挡物、阴影形状、高光边界、焦散落点。否则模型会把脸抹成均匀白亮，缺少真实层次。

**必须观察并写明**:
- 光源方向：左上、右上、正上、侧逆光等。
- 遮挡物：手指、头盔边框、镜片边缘、发丝、刀身等。
- 阴影形状：手指投影、头盔边框投影、斜切阴影、窄条阴影、块状阴影。
- 高光边界：过曝区域在哪里，边缘是硬切、柔和过渡还是棱镜彩边。
- 彩虹焦散：红黄蓝紫小光斑落在脸颊、鼻梁、眼周、手套或头发的具体位置。
- 光影主次：脸部可以过曝但不能完全无阴影；核心五官必须保留。

**稳定写法**:
```
强烈斑驳硬光从左上照入，白色手套和头盔镜片在脸上投下不规则手指阴影与弧形边框阴影，鼻梁和脸颊有硬边过曝白光，眼周、脸颊和手套上有红黄蓝紫分离的棱镜焦散小光斑，脸部不是均匀白亮，必须保留阴影切块和高光边界
```

**常见失败与修正**:
- 脸变成均匀奶白：新增"不是均匀柔光，必须有手指投影、镜片边框阴影、硬边过曝和局部暗块"。
- 彩虹光斑变成装饰彩点：新增"焦散光斑由镜片折射产生，落在脸颊/眼周/手套表面，跟随光源方向"。
- 真实感不足：新增"高对比硬光、遮挡投影、棱镜彩边、局部过曝与局部阴影同时存在"。

### 3.2 几何与裁切锁定

参考图拆解时，几何信息优先级高于风格词。若几何没有锁定，即使物件和材质正确，也视为拆解失败。

**必须先写入提示词的几何锁**:
- 宽高比：明确横版/竖版/方图，以及接近比例（例如3:4、4:5、9:16）。
- 镜头方向/相机轴线：相机从主体的左前方、右前方、正面、侧面、上方或下方看；脸朝画面哪一侧；眼睛视线是否直视镜头。必须独立成句，不能被"特写/近景/俯视"代替。
- 镜头尺度：极近景、近景、半身、全身；避免只写"特写"这种模糊词。
- 主体占比：脸、手、物件分别占画面大约多少；是否允许裁掉头顶、下巴、肩膀。
- 关键物件位置：锚点在上/中/下、左/右、是否跨过脸、遮住哪一部分。
- 边缘裁切：哪些物件贴边、露出几分之一、是否完整出现。
- 透视类型：平面压缩、长焦压缩、轻微俯视/仰视、禁止广角畸变。

**视角、镜头方向、镜头距离、主体占比必须分开写**:
- 视角只回答"从哪里看"（俯视、仰视、平视、侧视），不能替代镜头距离。
- 镜头方向回答"相机站在主体哪一侧看"以及"脸朝画面哪边"；例如"相机从角色右前方略高处看向左半张脸，脸朝画面右下，主眼斜向看镜头"。
- 镜头距离回答"离主体多近"（极近景、近景、中近景、半身、全身），必须单独写。
- 主体占比回答"主体在画面里多大"；复杂图必须写脸/头、手、前景物、身体或衣物分别占画面约多少。
- 裁切范围回答"画面截到哪里"；必须写是否裁掉头发、手臂、衣摆、前景物两端、背景文字边缘。
- 高角度俯视不等于远景。若参考图是俯视近景，必须写"高角度俯视近景/中近景，保持近距离，不要拉远成全身鸟瞰"。
- 若参考图人物没有完整铺满画面，禁止用"大面积衣物铺开、完整身体旋转"这类会诱导模型拉远的词；改写为"只保留局部衣物纹样，身体被边缘裁切"。
- 需要保真时使用视口坐标锁：脸中心约在画面(x%, y%)，脸高度约占画面H%，前景物宽度约占画面W%，背景文字占上方H%。坐标不必精确到像素，但必须能防止模型重新构图。

**常见失败与修正**:
- 参考图是3:4但生成成9:16：提示词必须前置"3:4竖版构图，不要9:16长图，不要过度拉长画面"。
- 参考图是装饰性平面特写但生成成写实大头照：必须写"图形化二次元面部、弱化真实鼻唇体积、不要写实皮肤和厚唇"。
- 参考图有边缘物件但生成被裁成一条边：必须写"左侧圆形物件露出约三分之一到二分之一，能看见圆孔结构"。
- 参考图脸部并非全屏但生成过度放大：必须写"脸部不要填满整个画面，保留上方头发、横向反射物和下方衣物区域"。
- 参考图是俯视近景但生成成远景全身铺陈：必须写"高角度俯视近景，不要鸟瞰远景；脸部高度约占画面12%-18%，不要低于10%；身体和衣物只保留局部裁切，不要完整展开全身"。
- 前景道具变得过大或抢镜：必须写"前景道具是对角线引导物，不是最大主体；道具宽度/高度限制在参考图相近范围；不要遮住过多人物脸部和身体"。

### 3.2.1 一发锁定闸门：反制模型默认审美

复杂参考图如果包含非对称紧裁、前景遮挡、强透视、文字压字、局部反射、镜面/玻璃/金属等高风险构图，直接写"华丽、清透、镜面、高级"会让模型自动整理成端正居中美人图、完整胸像、对称配饰或装饰玻璃板。生成提示词前必须先通过本闸门。

**必须前置的反制项**:
- 若参考图不是标准头像，必须写明"不是端正居中头像、不是完整美人肖像、不要左右对称、不要完整胸像"。
- 若有遮挡物，必须写明层级关系："遮挡物在人物/主体前方，是贴近镜头的前景平面，遮住哪个五官/身体部位，占画面多少"；禁止只写"中央有某物"。
- 若构图依赖不完整裁切，必须写明"顶部/底部/左右边缘裁切到哪里，哪些物件只露出局部，哪些物件贴边"；禁止让模型完整展示所有物件。
- 若视觉锚点只有一个，必须写明"唯一视觉锚点是X，其他配饰和背景必须受控、弱化、不能抢主体"。
- 若材质只应该落在某个物件上，必须写明"镜面/玻璃/金属/折射只落在X上，不要落到其他物件上"。
- 若反射、嵌入、印花容易混淆，必须二选一写死；例如"只能是镜面反射，不是印花，不是内嵌图案，不是装饰玻璃板"。禁止使用"反射或嵌入"这类二义性表达。
- 若参考图有大字、压字、海报文字或图形背景，必须写明文字的大小、位置、被主体遮挡关系、是否允许乱码；如果文字不是目标，应明确"不要新增可读文字"。

**常见默认审美跑偏与修正**:
- 生成变成端正漂亮头像：新增"非对称紧裁、偏心主体、脸部被前景物切割、主眼/副眼权重不同"。
- 生成变成完整胸像：新增"肩部只露底部一小段、不要完整胸像、不要展示完整服装"。
- 生成多了对称头饰/珠宝：新增"禁止对称发饰、禁止大型耳饰、禁止额外珠宝吊坠、禁止配饰抢主体"。
- 镜面物变成花纹板：新增"只能反射环境花影，不是印花、不是屏风、不是内嵌装饰图案"。
- 复杂海报变成普通人物插画：新增"背景图形/文字是构图骨架，必须位于主体后方并被主体局部遮挡"。

**输出前自检**:
如果提示词开头没有同时覆盖"比例、镜头方向、镜头距离、主体占比、遮挡层级、裁切边界、唯一视觉锚点、默认审美反制"，不得输出最终可复制提示词。

### 3.3 失败图反向归因

当用户提供生成结果并说差距大时，先做反向归因，禁止直接重写提示词。

按以下顺序列出差距和原因：
1. 几何差距：宽高比、裁切、镜头距离、主体占比、物件位置。
2. 视觉主次差距：哪个锚点变强/变弱，是否出现多锚点竞争。
3. 风格差距：是否从二次元图形化变成写实、3D、游戏CG、厚涂、照片感。
4. 色彩分布差距：参考图的大面积低饱和区、局部高饱和区是否被改变。
5. 材质差距：镜面、玻璃、水滴、金属、织物是否对应到正确物件。
6. 微细节差距：瞳孔、睫毛、花蕊、刀身反射、水滴高光、衣纹等是否缺失。
7. 提示词诱因：指出哪些正向词导致跑偏，哪些负向词缺失。

输出修正版时必须包含"删除/弱化词"和"新增锁定词"，而不是只给新的完整提示词。

### 3.4 工具升级判断

优先使用视觉观察和本地图片元数据完成拆解。若用户要求极高保真，或多次出现"元素对但构图/细节错"：
- 对本地图片先读取尺寸，确认真实宽高比；不要凭肉眼猜比例。
- 可用现有工具生成缩略对照图、局部裁切图、或放大检查图，临时文件只放在用户允许的临时目录。
- 若需要自动区域识别/分割/详细图像描述，可考虑 Florence-2、GroundingDINO、Segment Anything、Grounded-SAM、CLIP/WD14 tagger 等开源工具；使用前先判断成本和必要性，不要为简单单图拆解盲目安装重型依赖。

### 4. 逐项精确校验

**在生成提示词之前，必须逐项确认以下要素的准确性：**

| 校验项 | 确认要点 | 示例 |
|--------|----------|------|
| 光源方向 | 确认光源来自哪个方向（左上/右上/正上/侧光/逆光） | ✅ 光源来自左上，在头发和脸颊形成高光 |
| 光影类型 | 确认是硬光还是柔光，阴影是否柔和 | ✅ 柔和高光，微妙阴影（非硬阴影） |
| 主体颜色 | 精确描述主体颜色，避免笼统描述 | ✅ 纯白色头发（非灰色），左眼紫色带粉色反光 |
| 材质质感 | 确认材质类型（丝绸/金属/玻璃/布料等） | ✅ 黑色丝绸和服，镜面反射效果的刀 |
| 反射效果 | 确认是否有反射/折射效果 | ✅ 刀身清晰反射花朵和头发 |
| 构图方式 | 确认构图类型和视角 | ✅ 特写肖像，刀水平横跨下半脸 |
| 风格类型 | 精确匹配风格词库中的风格 | ✅ 写实数字绘画风格，结合动漫人物设计 |

**注意**: 如果发现任何要素描述不准确，必须重新观察图片，直到确认所有要素精准无误。

### 5. 分析视觉要素

系统分析以下方面：

**主体分析**:
- 主体内容（人物、动物、物体、风景）
- 主体特征（姿势、表情、服装、配饰）
- 主体数量及相互关系

**微细节分析（参考图任务必做）**:
- 对每个核心物件列出可见细节，不少于5项；例如花朵要写花瓣、花蕊、透明边缘、内部亮丝、碎片、露珠、光轨。
- 把"美、精致、清透"翻译成物理可见描述；例如"玻璃花"应拆成半透明花瓣、蓝白折射边、花瓣内发光脉络、白色尖锐高光、漂浮碎片。
- 标明哪些细节必须生成，哪些可自由变化。禁止把必须细节只放进"关键词"而不放进直接复制提示词。

**风格分析**（匹配style_library.md）:
- 艺术风格（印象派、超现实主义、动漫风格、数字绘画等）
- 摄影风格（肖像、风景、街头、电影感等）
- 文化风格（浮世绘、中国水墨、敦煌风格、国潮等）
- 数字风格（像素艺术、低多边形、故障艺术等）

**光影分析**:
- 自然光（黄金时刻、蓝调时刻、月光、极光等）
- 人工光（霓虹灯、钨丝灯、聚光灯、环形灯等）
- 光线方向（逆光、侧光、顶光等）
- 光影效果（剪影、镜头光晕、丁达尔效应等）

**色彩分析**:
- 色彩调色板（暖色/冷色、高饱和/低饱和）
- 具体色调（匹配style_library.md中的色彩参考）
- 对比度（高/低/中）

**构图分析**:
- 取景方式（黄金分割、三分法、居中构图等）
- 视角（正面、侧面、鸟瞰、仰视）
- 景深（浅景深虚化、深景深清晰）

**材质分析**:
- 表面纹理（光滑、粗糙、光泽、哑光）
- 材质类型（丝绸、皮革、金属、玻璃、毛发等）

**氛围分析**:
- 情绪基调（宁静、戏剧性、忧郁、充满活力）
- 环境效果（雾气、雨水、尘埃、闪光）

**艺术家参考分析**:
- 将风格匹配到知名艺术家（参考style_library.md）
- 使用"风格类似"或"受...风格影响"来引用艺术家

### 6. 小白友好补全规则

当用户没有参考图、描述不完整或不知道怎么表达时，不要直接输出一堆抽象风格词。先自动补全一版稳定默认方案，并告诉用户哪些地方是默认补全。

**默认补全建议**:
- 默认镜头：中近景或半身构图，主体清晰
- 默认光影：柔和定向光，避免强硬阴影破坏主体
- 默认构图：主体明确、背景不抢戏，必要时使用紧裁切
- 默认细节密度：主体高细节，背景中等细节
- 默认负面约束：低质量、畸形手、主体不清、过度装饰、风格混乱
- 默认平台：用户未指定时，优先给"GPT Images / ChatGPT Images 直接复制版"，不再默认给Midjourney版

**少问原则**:
- 信息缺失但不影响主要方向时，直接补默认值
- 信息缺失会导致完全不同画面时，只问一个关键问题
- 问题之后也要给一个"如果你不想选，我建议用..."的默认选项

当用户提供生成结果复盘时，必须输出：
- 跑偏点：逐条说明哪里不像
- 原因：指出是提示词缺失、权重过强、负面约束不足，还是模型默认审美导致
- 修正策略：哪些词提前、哪些词删减、哪些词加入负面
- 可直接复制的下一版提示词

### 8. 参考风格词库

分析时必须先查阅 `style_library.md` 获取基础术语；当画面有非泛化审美、罕见媒介、特殊材质、影像缺陷、亚文化造型、海报/产品/空间风格时，再查阅 `references/rare_style_library.json` 精确匹配稀有亚种风格词。

**艺术风格匹配**:
- 如果图片显示柔和笔触和光影强调 → "印象派，莫奈风格"
- 如果图片有梦幻、奇异的意象 → "超现实主义，达利风格"
- 如果图片是动漫风格带平涂 → "动漫风格，赛璐璐着色"
- 如果图片有霓虹灯和未来城市 → "赛博朋克"

**摄影风格匹配**:
- 如果图片有胶片颗粒和复古色彩 → "胶片摄影，柯达色调"
- 如果图片是黑白高对比度 → "黑白摄影"
- 如果图片有电影感灯光和构图 → "电影感摄影"

**光影匹配**:
- 如果是温暖柔和的日落/日出光芒 → "黄金时刻光线"
- 如果是暮光冷色调 → "蓝调时刻光线"
- 如果是逆光主体带光晕 → "逆光，剪影"

**艺术家匹配**:
- 动漫风格带详细背景和情感天空 → "新海诚风格"
- 奇幻风格带魔法世界 → "宫崎骏风格"
- 未来感镀铬机器人 → "空山基风格"

**稀有亚种风格匹配**:
- `references/rare_style_library.json` 含 620 条风格词，字段包括 `中文风格名`、`English prompt tokens`、`类别`、`视觉DNA / 关键词`、`材质/色彩/光线`、`容易翻车`、`补救提示`。
- 匹配时必须用视觉证据反推，不要因为词好看就加入。证据可以来自材质、色彩、镜头年代感、印刷缺陷、角色服装、空间气质、媒介工艺。
- 每次最多选择 1 个强主风格、0-1 个材质/表面、0-1 个光色或版式、0-1 个影像缺陷、0-1 个人物造型。过多强风格会让生成图摇摆。
- 如果风格词会威胁主体识别，必须吸收该条目的 `补救提示`，例如 `clear silhouette`、`recognizable subject`、`material only on surface`、`controlled grain`。
- 输出时给出 2-4 个候选风格及证据，最后只把最匹配的少量词写进直接复制提示词。

**七层组合语法**:
| 层级 | 数量 | 作用 | 反推时怎么用 |
|------|------|------|--------------|
| Base 主风格 | 1 | 决定画面大方向 | 只选一个，如电影、动画、摄影、工艺或数字媒介 |
| Surface 材质 | 0-1 | 增加触感和质地 | 只作用于表面，避免主体变成抽象纹理 |
| Light/Color 光色 | 0-1 | 强化氛围 | 和主风格冲突时以主风格为准 |
| Format 版式 | 0-1 | 海报、包装、卡牌、目录感 | 有文字时限制文字数量，避免乱码 |
| Space 场景 | 0-1 | 让主体落到具体空间 | 保持 foreground hero object 或 centered subject |
| Defect 缺陷层 | 0-1 | 胶片、CRT、复印、扫描等真实媒介感 | 强度不要超过 0.55，避免画质变脏 |
| Character/Fashion 造型 | 0-1 | 人像、IP、拟人产品妆造 | 只保留 1-2 个关键配饰 |

**新东方神怪/水墨语法**:
- 当图片或用户需求出现山海经、神怪、异兽、国风神话、工笔水墨、矿物颜料、留白烟云时，查阅 `references/shan-ze-style-grammar.md`。
- 最终提示词不要直接依赖在世艺术家姓名；改写为可生成的视觉语法：new-oriental mythic ink-and-color illustration, gongbi baimiao contour, iron-wire linework, mineral-pigment washes, ink-gray smoke-cloud negative space, xuan-paper or silk texture。
- 负面约束优先加入：no photorealism, no anime cel shading, no oil-paint impasto, no 3D render, no glossy plastic surface, no cyberpunk neon。

## 质量指南

1. **精确性**: 使用style_library.md中的精确术语，避免模糊描述
2. **完整性**: 覆盖所有视觉要素（主体、风格、光影、色彩、构图）
3. **专业性**: 使用行业标准术语
4. **可操作性**: 确保提示词可直接用于AI绘画工具
5. **特异性**: 尽可能识别并包含具体的艺术家、风格、技术
6. **相似度优先**: 复刻类任务中，构图、姿态、裁切和主体位置优先于风格词
7. **小白可用**: 输出必须包含直接复制版，并说明默认补全与防跑偏词
8. **参考图细节不能被概括词替代**: 如果参考图中有明确可见的花瓣、花蕊、指甲、掌纹、瞳孔、光轨、碎片、吊坠、布料褶边等，必须写入直接复制提示词。只写"精致、华丽、清透、高级"视为未完成拆解。
9. **几何不准就是拆解失败**: 宽高比、裁切、主体占比、关键物件位置不准时，即使材质和元素正确也不能算提示词有效。

## 跨平台提示词适配

### GPT Images / ChatGPT Images（默认）
- 使用中文自然语言描述，不使用Midjourney命令参数
- 用完整句子锁定画面比例、镜头方向、镜头距离、主体占比、裁切边界、遮挡层级、视觉焦点、光源方向、材质和细节密度
- 将负面约束写成"避免/不要/不要出现..."的自然语言边界，而不是纯tag堆叠
- 少用权重符号、版本号、采样器、模型参数；需要强调时用"必须突出..."、"画面核心是..."、"不要让...抢焦点"

### Midjourney
- 添加 "--ar X:Y" 设置宽高比
- 添加 "--v 6" 或 "--v 5" 指定版本
- 添加 "--q 2" 提高质量
- 使用简洁的逗号分隔格式
- 可以使用 "by [艺术家名]" 格式

### Stable Diffusion
- 使用逗号分隔格式
- 需要时使用 "BREAK" 分隔段落
- 同时包含正面和负面提示词
- 使用权重如 "(关键词:1.5)" 来强调

### Flux
- 更自然语言友好
- 可以使用描述性句子
- 较少依赖逗号分隔标签

### DALL-E 3
- 使用自然语言描述
- 更具描述性和叙事性
- 避免过多技术术语
- 注重视觉描述而非风格标签

## 重要注意事项

1. **必须查阅style_library.md** - 在生成提示词之前确保术语准确
2. **风格精准匹配** - 存在具体术语时不要使用通用术语
3. **包含质量标签** - 添加"杰作，最佳质量，超详细"等提升结果质量
4. **正确引用艺术家** - 使用"风格类似"或"受...风格影响"
5. **考虑平台差异** - 默认面向GPT Images / ChatGPT Images；Midjourney和Stable Diffusion需要另行调整参数与格式
6. **视频处理** - 通过分析关键帧处理视频，提供合并提示词和逐帧提示词
7. **不要只堆风格词** - 先锁画面骨架，再补风格、材质和质量词
8. **控制装饰过量** - 华丽、复杂、精细等词需要配合limited/controlled等边界词
9. **复盘要给下一步** - 分析差距后必须给可直接复制的修正版提示词
